ExtraSensory · context enrichment · RQ1b

Does the language model need coordinates, or meaning?

The LLM was weak at guessing where a person is from phone sensors alone. Feeding it raw GPS numbers barely helped — but feeding it a reverse-geocoded place name nearly doubled its accuracy and pushed it past the ML baseline. Its weakness was missing semantic location, not missing coordinates.

How to read it, honestly. C2 is an LLM + geocoder hybrid — a coordinate near “…State Beach…” makes beach easy, so the geocoder does real work. The finding is that the LLM correctly uses semantic location, not that it reasons brilliantly on its own. And this is not a clean LLM-beats-ML claim: ML was never given the coordinates, since raw location is nearly a per-person identifier (the source paper excluded it on purpose). Read it as: semantic location unlocks the LLM.
